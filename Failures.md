# Active Blockers and Failures

- DB-SVG cutover (authority). Live Admin publish and the published SVG gate still treat **disk** as release authority (`inventory/descriptors/`, `public/svg-catalog/`). Lifecycle audit remains filesystem (`results/admin/catalog-ops/`). Dual-write injects when Products DB is configured **and** R2 ListObjects succeeds; dead R2 does not roll back disk at the resolve gate. When dual-write is **enabled**, publish uploads honest immutable artifacts to R2 (descriptor JSON + SVG + real PNG with matching SHA-256 — not an SVG-hash stub for PNG) and writes DB revision + product pointer via `DrizzleSvgRevisionPersistence`. **Enabled dual-write with real artifacts still ≠ cutover:** Planner `svg-blocks` remains DB-aware with **disk fallback**; consumers must not assume DB/R2 is sole authority until contract items `DB-SVG-01`…`DB-SVG-20` in `docs/architecture/08-DATABASE-SVG-CONTRACT.md` are proven end-to-end. Next action: one production dual-write publish, then flip Planner published-SVG read and Admin gate to DB/R2-only (no disk override).
