# One&Only Website Target

Date: 2026-07-07

Status: target definition.

Source truth: live site at `https://oando.co.in`.

## What We Are Building

We are building the best possible website for One&Only.

One&Only is an office furniture and workspace solutions business.

The website must sell trust, products, planning, projects, and service.

It is a premium business website with serious operational modules behind it.

## Core Business

The website should support these business lines:

1. Ergonomic seating.
2. Workstations.
3. Tables.
4. Storage.
5. Soft seating.
6. Education furniture.
7. Workspace planning.
8. Institutional and enterprise projects.
9. Quotes and briefs.
10. Partner and admin workflows.

## Primary Goal

Turn a visitor into a qualified workspace enquiry.

The path should be simple:

1. Understand One&Only.
2. Browse products or solutions.
3. See proof through projects.
4. Plan or configure a workspace.
5. Request a quote or submit a brief.
6. Let the team manage the lead, project, assets, and follow-up.

## Positioning

One&Only should feel like:

1. A premium office-solutions company.
2. A planning-led workspace partner.
3. A reliable execution team for institutions and enterprises.
4. A local Indian business with national-quality presentation.
5. A company that understands durability, ergonomics, delivery, and after-sales support.

## Planner Target

The planner should make workspace planning easier.

Minimum target:

1. Open quickly.
2. Place furniture blocks.
3. Edit layout.
4. Undo.
5. Save.
6. Reload.
7. Export or share.
8. Create quote intent.
9. Produce no unexplained console errors.

Real Goal:

1. 2D floor planning.
2. 3D workspace preview.
3. Product-aware block library.
4. BOQ or item summary.
5. Admin-managed catalog.
6. Partner or sales-assisted planning.

## SVG Quality Target

SVG is critical because planner quality depends on block quality.

The SVG system should support:

1. Clean source definitions.
2. Schema validation.
3. Sanitization.
4. Deterministic compile output.
5. Versioning.
6. Preview.
7. Publish and rollback.
8. Runtime compatibility checks.
9. Traceability to source product or block.

SVG output should be:

1. Valid.
2. Minimal.
3. Scalable.
4. Safe.
5. Deterministic.
6. Fast.
7. Visually consistent.

## Admin Target

Admin should be a real operating console.

It should not be a collection of fragile routes.

It should manage:

1. Products.
2. Categories.
3. Projects.
4. Portfolio media.
5. SVG blocks.
6. Planner content.
7. Leads.
8. Customers.
9. Quotes.
10. Partner users.
11. Assets.
12. Publish queue.
13. Audit trail.

Hard rule:

Every admin action that changes business data needs role control, validation, failure behavior, and evidence.

## Visual Target

The website should feel premium, calm, exact, and business-grade.

It should not feel like a template.

Visual principles:

1. Strong editorial layout.
2. Real project imagery.
3. Clear product hierarchy.
4. Confident typography.
5. Consistent spacing.
6. Controlled motion.
7. High-quality dark and light surfaces where appropriate.
8. Mobile-first polish.
9. No random theme drift.
10. No visual values hardcoded in TSX.

## Quality Standards

| Standard | Use |
| --- | --- |
| WCAG 2.2 | Accessibility for public, admin, planner, and forms. |

## Non-Targets

These are not the target:

1. A carpet website.
2. A flooring website.
3. A generic landing page.
4. A planner demo with no business flow.
5. An admin shell with dead routes.
6. A catalog with no quote path.
7. A CRM that cannot support real follow-up.
8. A pretty site with hidden broken gates.

## Ideal End State

The ideal One&Only website:

1. Looks premium.
2. Explains the business fast.
3. Shows products clearly.
4. Proves real project capability.
5. Lets users plan workspaces.
6. Converts visitors into briefs and quotes.
7. Gives admins control without developer help.
8. Keeps SVG and planner assets clean.
9. Manages leads and project flow.
10. Ships with evidence, not guesses.

That is the target.
