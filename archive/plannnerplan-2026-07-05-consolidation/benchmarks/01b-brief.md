# Phase 01B Binding Feasibility Brief

## Accepted benchmark decisions

- Minimal hidden host: semantic shell, one canvas, compact command strip, live diagnostics, catalog-asset proof, error/reset boundary.
- One registered `draw-wall` command invoked by visible control and keyboard architecture.
- Plain feasibility visuals only; final design is deferred to Phase 05.
- Explicit preview, snap marker, committed geometry, and status layers.
- Endpoint candidate first; grid and 45-degree constraint after endpoint evaluation; Alt temporarily suppresses snapping.
- Escape, context-menu cancellation, pointer cancel, blur, and unmount must not commit geometry.
- Desktop proves pointer/keyboard draw; tablet proves compact pointer/touch capability; small viewport declares limited editing.

## Rejected for Phase 01B

- Donor visual parity.
- Final toolbar/panel design.
- Docking, inventory browsing, 3D, onboarding, production persistence UI, or route replacement.
- Competitor-specific colors, icons, marker geometry, or screen composition.

## Acceptance

Phase 01B authorizes full design work only after React input/render, pure transaction/undo, cancel safety, canonical units, migration report, asset identity/fallback, latency, bundle, license, and no-Svelte evidence pass.
