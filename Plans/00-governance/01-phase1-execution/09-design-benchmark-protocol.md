# World-Standard Design Benchmark Protocol

## Mandate

The current planner UI is not the design baseline. The donor (`open3d-floorplan/`) is functional/behavioral reference only. The React planner requires a fresh design approach at the standard of leading current spatial-design, CAD, floor-planning, and professional creative tools.

Do NOT reproduce donor's visual composition, panel arrangement, styling, or information hierarchy unless current evidence independently supports it.

## Mandatory benchmarker agent

Before any phase or slice that changes UI, UX, drawing workflows, toolbars/panels, inventory presentation, visual design, onboarding, 2D/3D transitions, or export workflow:

1. Summon a dedicated benchmarker/research agent.
2. Require current web research using direct product/help sources.
3. Compare at least five relevant leading products:
   - 2 floor-planning/interior-design products.
   - 1 CAD or spatial-authoring product.
   - 1 professional creative-tool interface (Figma/Sketch/etc).
   - 1 product strong in large-catalog or asset-library UX.
4. Review desktop and tablet behavior.
5. Separate observed facts from recommendations.
6. Record dated citations with access date.
7. Save under `05-benchmark-foundation.md`, `06-benchmark-delivery.md`, or `07-benchmark-governance.md`. See `10-review-workflow.md`.
8. Primary agent reviews, rejects imitation and weak suggestions, and incorporates accepted decisions into binding plans/design records.

Existing reports are background only — they do not satisfy a future execution benchmark when materially stale.

## Required research dimensions

- First-run, empty state, templates, progressive teaching.
- Canvas-to-chrome ratio and visual hierarchy.
- Drawing-tool discovery, feedback, snapping, measurements, cancellation.
- Selection, properties, contextual controls, command search, shortcuts.
- Constrained docking, movement, resizing, presets, reset, responsive adaptation.
- Inventory taxonomy, search ranking, filters, variants, previews, recents, favorites.
- 2D/3D continuity, loading, camera state, missing assets, performance.
- Visible save/offline/recovery/conflict truth.
- Import/export lifecycle, preflight, progress, preview, history, stale output.
- Accessibility, keyboard operation, focus, contrast, non-canvas structure.
- Visual language, density, typography, iconography, color, motion.

## Design output required before implementation

Primary agent must produce or update a binding phase design brief containing: target users and top tasks, information architecture, default desktop and tablet layouts, viewport capability statement, toolbar/panel grammar, command and drawing-tool hierarchy, inventory navigation model, key screen and state matrix, accessibility behavior, performance budgets, accepted inspirations with reasons, rejected inspirations with reasons, intentional differences from donor/competitors, testable acceptance criteria.

No UI implementation begins from screenshots alone or from an agent's unreviewed suggestions.

## Fresh-design acceptance

Acceptance requires:
- Task completion and drawing correctness.
- Clear, accessible, responsive structure.
- Professional visual coherence.
- Excellent default layout before customization.
- Inventory usable at scale.
- Recoverable toolbar/panel customization.
- No donor visual debt carried forward without explicit justification.
- Evidence that the result is internally coherent, not a collage of competitor patterns.

## Anti-copy rule

Borrow interaction principles and validated patterns. Do NOT copy proprietary colors, wording, icons, exact geometry, screen composition, assets, or trade dress.

## Stale-evidence policy

Immutable archived reports live in `archive/Plans/2026-07-05_phase1-execution/benchmarks/`. Do not edit in place.

1. Do NOT edit the original file.
2. Add a dated correction note in the phase evidence file referencing the original report and stating what changed.
3. If acceptance decisions depend on stale findings, create a new dated benchmark report.

This preserves the historical record while ensuring current decisions are based on current truth.
