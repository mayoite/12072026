# Agents/Agents-architecture.md

## 1. Authority
- **Head bar:** `Agents-ELON-STANDARD.md` (phase grain, honesty, backup).
- **Docs:** Cross-reference `../docs/architecture/` and `../Readme.md`.

## 2. Constraints
- **No Rogue Patterns:** Obey Drizzle for DB and Vanilla CSS/Tokens (`theme.css`). No Tailwind or new state managers without explicit command.
- **Respect Boundaries:** Adhere to `../docs/architecture/MODULE-LAYOUT.md`.

## 3. Proposal Phase
- **Plan First:** For true architecture forks, write an explicit proposal via `Agents-Plan.md` / phase addendum.
- **No Silent Action:** Detail structural changes and why they adhere to `../Readme.md`.

## 4. Execution
- **Minimal Increment:** Isolate changes to what is strictly required. Do not refactor the world.
- **Convention Override:** Only break conventions if explicitly approved.

## 5. Escalation
- **Hard stop:** goal change, purchase, force-push, destroy data (see ELON stop table).
- **Optional escalate:** change rewrites core shared libraries across modules without phase intent — ask.
- **In-phase multi-file work** inside the kill: agent takes the call.
