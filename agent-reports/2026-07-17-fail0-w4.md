## Root cause

In `publishActionMessages.admSvg15_17.test.tsx`, the `next/dynamic` mock did:

```ts
React.useEffect(() => {
  props.onDocument?.(/* studio svg + rect */);
}, [props]); // props is a new object every parent render
```

`onDocument` Ôćĺ parent `setState` Ôćĺ re-render Ôćĺ new `props` Ôćĺ effect again Ôćĺ **infinite render loop** Ôćĺ worker OOM. That is hang + suite-level FAIL, not a product bug.

Secondary bug on the success path after the loop was fixed:  
`useRouter` mock only had `refresh`. Success publish calls `router.replace(...)`. That threw, catch block overwrote success feedback with failure ÔÇö so ADM-SVG-17 success assertion failed.
