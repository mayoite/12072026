## Mission

Sample CSRF + rate-limit matrix on mutation routes. Planner handoff already green; extend admin ops + site forms unit/static proof.

## What changed

### `site/scripts/audit-api-route-safety.mjs`

- Scans all `app/api/**/route.ts` mutators (POST/PUT/PATCH/DELETE).
- **New:** `--matrix` TSV + `--json` includes full matrix rows (surface, csrfReq, csrf, rate, auth, rejection header, error code).
- **New:** public/site form mutators must keep rate limits (`missing-public-rate-limit`).
- **New:** CSRF required for `ai-*`, `ai/`, `audit` (session-backed mutators).
- **New:** wrong CSRF codes include `INSUFFICIENT_PERMISSIONS` (not only `INVALID_INPUT`).
- **New:** optional-CSRF paths that still implement CSRF must set `x-csrf-rejected`.
- **New:** honeypot warn on public `customer-queries` POST.
- Removed `audit` from `CSRF_OPTIONAL` (authenticated write → CSRF required).
