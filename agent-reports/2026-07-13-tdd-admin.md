## Verdict (brutal)

These suites are mostly **tests-after / checklist lock**, not strict red-green TDD.

They are **not hollow overall**. Pricing service/store/governance tests exercise real outcomes with injectable stores and temp dirs. Catalog manager utils are strong pure-function proof.

Several asserts would still pass if production were broken in specific ways (boolean helpers always true, UI not wired to constants, residual archive never retaining). Residual SVG files are edge-case patches; core archive retention lives in phase08, not residual.

**Isolation: good** for file-backed work. Canonical price-book seed and catalog files are not mutated when `dir` is injected.

## 1. TDD compliance

### Tests-after vs behavior-first

**Tests-after (dominant):**

- Headers cite ADM-* checklist IDs (`ADM-PRICE-01/02/03`, `ADM-ROLE-01`, `ADM-AUDIT-01`, residual “beyond phase08”).
- Suites document finished modules with broad happy + deny matrices.
- Residual files exist because phase08 left gaps, not because a red test drove the feature.

**Behavior-first attributes (present, good):**

- Prefer outcomes over spies: **no `vi.mock` in pricing unit tests**.
- Injectable `PriceBookStore`, clocks, `dir`, lock `sleep`/`now`.
- Fail paths assert **error substrings** and often **post-state** (version status, audit actions).
- Catalog utils assert exact payloads and throw messages, not “function was called”.

**Strict red-green proof: weak.** Nothing in-repo shows failing tests written first. Treat as post-hoc specification tests with above-average rigor.

### Hollow asserts

| Location | Issue | Severity |
|----------|--------|----------|
| `quotePriceBookPin.test.ts` | `"quoteId" in pinResult` is a weak success shape check | Medium |
| `quotePriceBookPin.test.ts` | `reproduciblePinnedTotal(...).toBe(true)` only; no false case | High |
| `quotePriceBookPin.test.ts` | `quoteStillValidAgainstBook` only true path | High |
| `descriptorLock.residual` stale acquire | Asserts `ok` + release; does not assert lock file content/mtime after reclaim | Low |
| `descriptorArchive.residual` | `retainDescriptorArchive(..., 1)` only proves no-op; not copy/evict | High *for residual alone* |
| `priceBookContract` currency | Digit strip + `/₹\|INR\|Rs/i` — loose on grouping/locale | Low |
| `adminNav` | Shape checks `typeof item.icon === "object"` — any object passes | Low |
| Early `if (!ok) return` after `expect` | TypeScript narrowing only; **not hollow** if `expect` runs first | OK |

`toBe(true)` / `toBeNull()` used after real domain checks (roles, statuses, file contents) is fine. Hollow pattern is **boolean-only** without side-effect or inverse cases.

### Isolation (temp dirs / catalog)

**Good:**

- `priceBookFileStore`, `priceBookAdmin.server`, governance audit: `mkdtempSync` + `rmSync` in `finally` / `afterEach`.
- Residual archive/lock: per-test temp workspace; no canonical `block-descriptors/` writes.
- Service tests: pure memory store; no disk.
- Comments state “never touches product seed”; injectable `dir` backs that up.

**Acceptable read-only product paths:**

- `emitPriceBookContract` / `quotePriceBookPin` read `features/planner/admin/pricing/fixtures/linear-desk-2026-q3.json` via `process.cwd()`.
- Risk: cwd must be `site/`. Fail loud if wrong; does not mutate catalog.

**Not in residual but adjacent risk:** other admin SVG tests read live `block-descriptors/` for fixtures. Residual suite does not.

## 2. Tests that would pass if production were broken

### Critical / high

1. **`reproduciblePinnedTotal` always returns `true`**  
   - File: `quotePriceBookPin.test.ts`  
   - Only asserts `toBe(true)`. A stub `return true` keeps the suite green.

2. **`quoteStillValidAgainstBook` always returns `true`**  
   - Same file. No case where pinned version is removed from the book.

3. **`pinQuoteAtBookVersion` failure branches dead**  
   - Missing version and non-pinable status (`retired` / `rolled_back`) untested.  
   - Production returns `{ ok: false, error }`; tests only happy path.

4. **`AdminPriceBookPageView` action / fetch wiring**  
   - Unit test only renders static `initialContract` presentation (currency, unavailable SKU).  
   - Broken `browserApiFetch`, confirm flow, role gating in the component still pass unit tests.  
   - Server action matrix is covered separately; **UI glue is not**.

5. **`adminNav` constants vs shell wiring**  
   - Hardcoded href lists lock `adminNav.ts` only.  
   - Shell/layout ignoring `ADMIN_NAV_GROUPS` still passes unit tests.

6. **Residual `retainDescriptorArchive` always no-op**  
   - Residual only checks `liveN <= 1`.  
   - Real retention is covered by `phase08-persistence.test.ts` (seven saves → archive `[2,3,4,5,6]`). Residual alone is a false sense of archive proof.

7. **Contract emission failure after successful save**  
   - `priceBookService` saves, then `emitPriceBookContract`; on emit fail returns `ok: false` **without restore** (unlike history-loss path).  
   - Tests assert error string only; do not assert store rolled back.  
   - Broken “fail closed on emit” still green.

### Medium

8. **`emitPriceBookContract` returns `null` for empty book/family** — only via service “contract emission” strings, not direct unit cases for dropped versions (blank `versionId`, bad currency).

9. **`lineTotalMinor` edges** — NaN price covered; `quantity <= 0` and free `unitPriceMinor === 0` not asserted.

10. **`formatPriceBookMinorSecondary`** — exported, used by display path, **no direct unit test**.

11. **Deny paths that do not assert store immutability** — e.g. non-approver activate asserts error but not that versions stayed approved.

### Low / not hollow

12. Type-narrowing early returns after failing `expect` — correct pattern.  
13. `normalizeRules` mutation isolation — solid (copy identity checks).  
14. File store corrupt JSON / bookId mismatch → null — would fail if reader always returned junk objects.

## 4. Good patterns (keep)

1. **Temp-dir isolation with cleanup** — file store, admin server, audit, residual SVG.  
2. **Injectable ports** — `PriceBookStore`, lock `now`/`sleep`, archive `dir`.  
3. **Outcome-oriented service tests** — role deny, status transitions, demote prior active, rollback fallback active, history-loss restore (`saveCount === 2`).  
4. **Corrupt input handling** — bad JSON price book, audit non-JSON lines, bookId/file mismatch.  
5. **Commercial invariant** — missing price → unavailable, never zero display; invalid emit rules dropped.  
6. **Audit resilience** — commercial action still succeeds if audit append path is a directory.  
7. **Catalog manager utils** — round-trip draft ↔ payload, sizing-type JSON gates, designer GLB rejection, exact error strings.  
8. **plannerAdminLinks** — encoding, page/status/search omit rules; brittle strings that break on intentional contract change (desired).  
9. **No catalog mutation** in scoped suites when dirs are injected.  
10. **Phase08 + residual split** — residual fills clear/sweep edges; phase08 owns retention and lock-busy.

## 8. Summary for owner

**Violations / debt**

- Quote pin suite is the weakest TDD link: boolean success without inverse cases.

- Admin nav + price book page view are data/presentation locks, not end-to-end Admin behavior.  
- Emit-fail-after-save is under-specified in tests and may hide a product consistency bug.  
- Style is **tests-after checklist proof**, not red-green TDD.

**Good patterns**

- Temp dirs, injectable stores, no pricing mocks.  
- Rich service status/role/history matrix.  
- Catalog manager utils are model pure-function TDD-quality proof.  
- Missing price never renders as zero.  
- File corruption and audit edge cases treated as real behavior.

**Overall:** usable regression suite for Admin pricing core and catalog draft utils. Do not claim strict TDD. Do not use residual SVG tests alone as archive/lock acceptance.
