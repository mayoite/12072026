# Issues 01 — Code (cross-cutting)

**Source:** Earlier major-issue audit.  
**UI verified:** No.  
**One sentence each.**

1. Products DB is not released SVG authority; disk (`inventory/descriptors/` + `public/svg-catalog/`) still owns publish and most reads.  
2. Admin DB dual-write uses a stub revision payload, so dual-write “success” is not real revision truth.  
3. Publish is not one atomic transaction for artifact + revision + product pointer + audit.  
4. Catalog lifecycle and audit still live in gitignored `results/admin/catalog-ops/` files, not a durable store.  
5. Price books are filesystem JSON, not governed commercial database records.  
6. Multiple BOQ builders coexist without one calculation authority.  
7. Planner keeps parallel trees (`project/`, `catalog-api/`, `cloud-store/`) that duplicate catalog and persistence.  
8. CRM Admin surfaces are localStorage demo, not production customer infrastructure.  
9. Site catalog can fall back to local fixtures when DB is unset, so public pages can show non-released inventory.  
10. Conversion funnel events are defined but mostly unwired beyond `PLANNER_ENTRY`.  
11. Security control matrix and production-like auth proof without bypass are still open.  
12. Admin unit coverage was last measured far below an 80% floor and was not re-proven as full-tree PASS.  
13. Playwright Admin suites run under `DEV_AUTH_BYPASS`, so production auth behavior is not proven.  
14. There is no CI hash gate that blocks tests from mutating canonical `inventory/descriptors/`.  
15. Docs and feature maps still drift from live paths; code wins until cutover is proven.  
16. Planner validation is thin: axis-aligned overlap stub and empty compliance check.  
17. Review links, named immutable project revisions, and Send-to-Oando handoff are not end-to-end complete.  
18. Rate limits are not proven on a distributed production backend for expensive routes.  
19. Tool output under `results/` is easy to mistake for product PASS; it is not.  
20. The full Admin → Site → Planner → branded BOQ → Oando loop is not browser-proven as one journey.
